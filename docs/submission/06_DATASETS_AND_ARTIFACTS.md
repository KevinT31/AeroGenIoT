## Dataset Reality

### What Exists Now

The repository currently includes two CSV files that are useful as partial telemetry samples:

- `pc_hybrid_telemetry_log.csv`
- `pc_simulated_hybrid_telemetry_log.csv`

These are legitimate sample telemetry artifacts and should be considered for inclusion in the regional submission ZIP.

### What Does Not Yet Exist in the Repository

The repository does not yet include sanitized exports from the production MySQL operational tables. For a stronger regional submission, add:

- one sanitized sample from the telemetry compatibility table
- one sanitized sample from each AI output table

Recommended export size:

- 100 to 500 rows per sample
- remove credentials and internal-only metadata
- preserve the columns required to demonstrate the end-to-end flow

## Model Artifact Reality

The repository demonstrates the operational AI integration path but does not include:

- `.pkl`
- `.joblib`
- `.onnx`
- `.pt`
- `.h5`

or similar weight or serialized model files.

## Logs Reality

- ModelArts job output bundles
